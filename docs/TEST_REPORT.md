# 测试报告（飞书 → FoodTalks）

## 范围

- 入口一致性：点击图标打开小弹窗
- 主流程闭环：提取 → 进度条完成 → 复制（可重复）→ 新标签登录
- 失败恢复：凭据缺失 / 权限不足 / 图片拉取失败 / 登录缺失
- 单元测试：状态机、共享 HTML 处理、错误映射、FoodTalks URL 判断

## 当前自动化

执行：

```bash
npm test
```

覆盖：

- 飞书提取核心逻辑
- FoodTalks 发布 API 适配
- URL 判断与标题归一化
- 平台模块结构完整性

## 手工验收清单（建议）

1. 点击扩展图标是否打开 `popup.html`
2. 输入合法飞书链接后是否可完成提取
3. 提取时进度条是否从 0 到 100 且无文字
4. 提取后按钮是否变为“点击复制”
5. 首次复制后按钮是否变绿色“已复制”，且再次点击仍可复制

## 已知限制

- 自动发布能力仍属 Beta，页面结构变化可能导致选择器失效。
- 默认推荐流程仍是半自动人工确认，以保证可控性和稳定性。
